This demo shows you how you can search word phrases in a PDF document.

The single words are not compared to single search terms but the words are found by using regular expressions on the words result instance. The current regular expression matches parts and not whole words. The search term "word another thing" will e.g. end in following regular expression "/word\sanother\sthing/ui".

This logic is build into this demo. For sure you can use any regular expression in your own implementation.

To keep text columns together and to support dehypen logic the word group strategy is used.
